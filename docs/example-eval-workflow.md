# Example workflow — dataset evaluation

[`example-eval-workflow.json`](example-eval-workflow.json) is an importable n8n
workflow that runs a full **LLM evaluation loop** using Langfuse datasets and a
real OpenAI call. It is the practical use case behind the Dataset API: datasets
are the central building block for evaluating LLM output in Langfuse.

## What it does

```
Manual Trigger
  → Set Run Config            (datasetName + a unique runName)
  → Create Dataset            (once)
  → Build Test Cases          (fans out into N cases: question + expected answer)
  → OpenAI Chat               (answers each question)
  → Grade                     (exact-match compare → score 1/0, carries ids forward)
  → Create Trace              (per case: logs the question + answer)
  → Create Dataset Item       (per case: upserts the item by a fixed id)
  → Create Run Item           (per case: links item + trace into the run)
  → Score Result              (per case: attaches the grade to the trace)
  → Get Dataset Run           (reads the run back)
```

Each test case flows through as its own n8n item. The Langfuse node sets
`pairedItem`, so every node downstream can read its per-case values from
`$('Grade').item` without the cases getting mixed up.

## The evaluation loop

This mirrors how Langfuse dataset runs work:

1. A **dataset** holds reusable test cases (**dataset items**: input + expected output).
2. A **dataset run** is one pass of your app over those items.
3. Each **dataset run item** links a dataset item to the **trace** produced when
   your app processed that item.
4. **Scores** on those traces turn the run into a measurable evaluation.

## How to import

1. Install the community node `n8n-nodes-langfuse-studio` (>= 1.3.0).
2. In n8n: **Workflows → Import from File** and select `example-eval-workflow.json`.
3. Open each **Langfuse** node and select your **Langfuse API** credential
   (placeholder `REPLACE_LANGFUSE_CRED`).
4. Open **OpenAI Chat** and select your **OpenAI** credential
   (placeholder `REPLACE_OPENAI_CRED`); adjust the model if needed.
5. Click **Test workflow**, then open the dataset run in Langfuse to see the
   linked traces and `exact_match` scores.

## Design notes

- **Synchronous vs async.** Dataset Public API writes (dataset, item, run item)
  are synchronous, so `Get Dataset Run` finds the run immediately. Scores are sent
  through async ingestion, so they may appear in the run a moment later.
- **Deterministic ids.** `Build Test Cases` generates a stable `itemId` and
  `traceId` per case, so `Create Dataset Item` upserts (rather than duplicates)
  and the run item links to the exact trace.
- **No single point of failure.** Every Langfuse node has *Continue On Error*, so
  a failed step never stops the run.
- **Source of truth.** The JSON is generated by
  [`scripts/gen-eval-workflow.mjs`](../scripts/gen-eval-workflow.mjs); edit the
  script and re-run `node scripts/gen-eval-workflow.mjs` rather than hand-editing
  the JSON.
